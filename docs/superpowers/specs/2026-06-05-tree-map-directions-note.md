# FruitySplash Tree Map Directions

Status: active exploration note

## Current build direction

Primary path for the next map-system iteration:

1. Straight vertical climb
   - One main trunk
   - Clean upward progression
   - Fastest version to build and tune
   - Best foundation for an infinite tree progression system

## Parked expansion ideas

Keep these for later versions after the straight-climb base is working:

2. Branch hopping
   - Main trunk with side branches
   - Optional reward nodes, event nodes, or short detours
   - Good for adding choice without making the base path confusing

3. Spiral climb
   - The path wraps around the tree as the player ascends
   - Strong magical feeling and better scenic variety
   - Good when the map needs more spectacle and motion

4. Cluster climb
   - Large fruit platforms or canopy zones
   - Several levels live inside one cluster before the player moves upward
   - Good for chapter-like pacing without returning to heavy hand-made screens

## Why this direction is promising

- Keeps the fruit-and-tree identity
- Avoids feeling like a direct Candy Crush copy
- Supports infinite progression naturally
- Can reduce content-production time by reusing one climb system
- Lets visual themes evolve by height, season, weather, fruit type, or magical biome

## Working principle

Build version 1 as a reusable vertical tree-lane system first.
Then layer branch, spiral, or cluster logic on top later instead of designing every map as a custom screen.
